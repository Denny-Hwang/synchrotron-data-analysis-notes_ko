## 요약 (TL;DR)

토모그래피 재구성, DL 기반 노이즈 제거(denoising), 자동화된 분할(segmentation)을 Argonne의 HPC 인프라에서 실행되는 단일 실시간 파이프라인으로 통합하는 엔드투엔드 AI+HPC 워크플로우로, 빔라인 검출기에서 스트리밍되는 마이크로-CT 데이터를 1분 미만의 총 지연 시간으로 처리한다.

---

## 배경 및 동기

APS 2-BM과 같은 고플럭스(high-flux) 빔라인에서의 방사광 마이크로 컴퓨터 단층촬영(마이크로-CT)은 1 GB/s를 초과하는 속도로 데이터를 생성한다. 전통적으로 데이터는 디스크에 기록되고, 컴퓨팅 클러스터로 전송된 후, 오프라인으로 처리되는데 -- 이 워크플로우는 수집과 사용 가능한 3D 분할 체적 사이에 수 시간에서 수 일의 지연 시간을 도입한다. 이러한 지연은 실험자에게 실시간 피드백을 방해하여, 측정 훨씬 후에야 데이터 품질을 확인하거나 실험 파라미터를 조정할 수 있다. 저자들은 빔라인 데이터 수집을 Argonne Leadership Computing Facility (ALCF) 자원에 직접 연결하는 스트리밍 파이프라인을 개발하여, 재구성(TomoPy), DL 노이즈 제거(TomoGAN 변형), 분할(U-Net)을 단일 토모그래피 스캔 회전의 시간 창 내에서 실행한다.

---

## 방법

1. **데이터 스트리밍**: 프로젝션 이미지가 전용 100 Gbps 네트워크 링크를 통해 Globus 데이터 패브릭을 경유하여 검출기에서 ALCF로 스트리밍됨. 메시지 큐(ZeroMQ)가 데이터 가용성 알림을 조정.
2. **재구성 단계**: TomoPy가 수신 사이노그램에 대해 여과 역투영(filtered back-projection, FBP) 또는 gridrec을 수행. 병렬성을 위해 다중 CPU 노드에 분산되어, 2048x2048x2048 체적의 재구성을 20초 미만으로 달성.
3. **노이즈 제거 단계**: TomoGAN 파생 U-Net 모델이 NVIDIA V100 GPU에서 실행되어 재구성된 슬라이스의 저선량 노이즈를 억제. 생산자-소비자(producer-consumer) 큐를 통해 재구성과 동시에 처리가 겹침.
4. **분할 단계**: 사전 훈련된 U-Net이 노이즈 제거된 체적을 재료 상(기공 공간, 고체 매트릭스, 개재물)으로 분할. 모델은 유사한 시편 클래스의 수동 주석된 마이크로-CT 슬라이스에서 훈련됨.
5. **오케스트레이션**: Apache Airflow 기반 워크플로우 매니저가 세 단계를 조정하고, 오류 복구를 처리하며, 자원 활용률을 모니터링. 단계 수준 병렬성이 파이프라인 처리량이 검출기 프레임 레이트와 일치하도록 보장.
6. **시각화**: 재구성, 노이즈 제거, 분할된 슬라이스가 실시간 품질 관리를 위해 빔라인에서 접근 가능한 웹 기반 뷰어로 스트리밍됨.

## 강점

- 검출기에서 분할된 체적까지의 완전한 엔드투엔드 파이프라인을 시연하여, 실시간 방사광 토모그래피 분석이 기술적으로 실현 가능함을 증명.
- 1분 미만의 지연 시간이 실험 내 의사결정을 가능하게 함: 운영자가 다음 시료 회전 전에 데이터 품질을 확인하고 스캔 파라미터를 조정할 수 있음.
- 생산자-소비자 아키텍처가 단계들을 시간적으로 겹치게 하여, 처리량과 GPU/CPU 활용률을 극대화.
- 처음부터 구축하지 않고 확립된 커뮤니티 지원 도구(TomoPy, Globus)를 사용하여 유지 보수성 향상.
- 다른 시설에 실질적인 인프라 안내를 제공하는 네트워크 대역폭 요구 사항 정량화.

## 한계 및 격차

- 전용 100 Gbps 네트워크와 다중 노드 HPC 할당이 필요하며, 이는 모든 시설이나 항상 이용 가능하지 않음.
- 분할 모델이 특정 재료 클래스에 대해 사전 훈련됨; 새로운 시편 유형으로의 일반화에는 주석된 데이터를 사용한 재훈련이 필요.
- 오류 처리가 높은 수준으로 기술되었으나, 실패 조건(네트워크 중단, GPU 장애, 손상된 프로젝션)에서 엄격하게 테스트되지 않음.
- 노이즈 제거 모듈이 단순화된 TomoGAN 변형; 최첨단 대안(자기 지도 학습 방법, 확산 모델)과의 비교 없음.
- 처리된 출력에 대한 데이터 출처(provenance), 버전 관리, 메타데이터 관리에 대한 논의 없음. 이는 과학적 재현성에 중요.

## APS BER 프로그램과의 관련성

본 논문은 BER 프로그램의 실시간 분석 인프라에 대한 시스템 수준 템플릿을 제공한다:

- **파이프라인 아키텍처**: BER 프로그램은 토모그래피뿐만 아니라 다양한 측정 기법에 걸친 자체 다단계 분석 파이프라인에 생산자-소비자, 단계 겹침 설계를 채택할 수 있다.
- **ALCF 통합**: 여기서 시연된 Globus 기반 데이터 패브릭과 ALCF 자원 할당 패턴은 BER 프로그램의 GPU 추론 및 훈련을 위한 계획된 ALCF 사용에 직접 적용된다.
- **지연 시간 벤치마크**: < 60초 엔드투엔드 지연 시간이 BER 프로그램의 토모그래피 파이프라인 목표를 설정; APS-U의 더 높은 데이터 속도에는 추가 최적화가 필요.
- **피드백 루프**: 웹 기반 시각화 구성 요소가 BER 프로그램의 운영자 대시보드의 출발점이지만, 자동화된 의사결정 역량으로 확장되어야 한다.

## 실행 가능한 시사점

1. **생산자-소비자 스테이징 채택**: 단계 수준 병렬성 및 분리를 위해 ZeroMQ 또는 Kafka 메시지 큐를 사용하여 BER 프로그램의 분석 파이프라인 구현.
2. **Bluesky/Tiled로 업그레이드**: 맞춤형 데이터 스트리밍 레이어를 메타데이터가 풍부한 시설 표준 데이터 플로우를 위해 Bluesky 이벤트 모델과 Tiled로 교체.
3. **자기 지도 학습 노이즈 제거**: 쌍 훈련 데이터의 필요성을 제거하기 위해 지도 학습 TomoGAN을 Noise2Void 또는 유사한 자기 지도 학습 방법으로 교체.
4. **자동 QC 추가**: 파이프라인 출력이 임계값 이하로 저하될 때 경보를 트리거하는 자동 품질 관리 지표(PSNR, SSIM, 에지 선명도) 통합.
5. **APS-U 데이터 속도로 확장**: 예상 APS-U 데이터 속도(5~10 GB/s)에서 파이프라인을 프로파일링하고, 아키텍처 변경이 필요한 병목 식별.
