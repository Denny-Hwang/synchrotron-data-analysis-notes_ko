# Release Notes — explorer-v0.3.0

**Phase C: GitHub Pages static mirror**

## Summary

Ships a static HTML mirror of the Streamlit explorer to GitHub Pages so
the full 3-cluster IA and every note page is browsable without a local
Python runtime. The mirror shares the IA mapping, note parser, and CSS
with the Streamlit app and is regenerated by CI on every push to `main`.

## What's New

- **Static site generator** (`scripts/build_static_site.py`): Reads notes
  via `explorer/lib/notes.py`, groups them via `explorer/lib/ia.py`, and
  emits a full HTML site that mirrors `app.py`, `1_Discover.py`,
  `2_Explore.py`, `3_Build.py`, and the note detail view.
- **Reused Streamlit assets**: `explorer/assets/styles.css` is inlined
  into the generated `assets/styles.css` so visual tokens (ADR-005,
  DS-001) cannot drift.
- **New `.github/workflows/pages.yml`**: Installs
  `scripts/requirements.txt`, builds `site/`, uploads the artifact, and
  deploys via `actions/deploy-pages`. Triggers on changes to any note
  folder, `explorer/**`, the generator, the wireframes, or the workflow.
- **Wireframes preserved**: `docs/02_design/wireframes/html/*` is copied
  to `site/wireframes/` and a new `site/wireframes/index.html` lists the
  three existing mockups — no previous Pages URL breaks.
- **Note images mirrored**: All images in note folders (currently 21 in
  `09_noise_catalog/images/`) are copied so markdown image references
  resolve on the static site.
- **Markdown link rewriting**: Relative `.md` links in note bodies are
  rewritten to `.html` so inter-note navigation works on Pages.
- **Per-note metadata aside**: Mirrors the Streamlit metadata panel
  (beamlines, modality, tags, related publications/tools, cluster) and
  adds a direct "Source" link to the canonical `.md` on GitHub.
- **`404.html` + `.nojekyll`**: Friendlier not-found page; disables
  Jekyll so note filenames with underscores are served as-is.

## Sync contract

`CLAUDE.md` Invariant #9 and `docs/03_implementation/github_pages_sync.md`
require that any change to `explorer/` ship with the matching generator
change in the same PR. CI rebuilds the site automatically — there are no
hand-edited static files.
