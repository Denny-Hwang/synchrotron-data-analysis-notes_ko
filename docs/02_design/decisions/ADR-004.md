# ADR-004: 8 Folders → 3 Task Clusters IA Mapping

## Context

The repository has 9 top-level folders (01–08 plus 09\_noise\_catalog). Presenting all 9 as navigation choices exceeds the cognitive load recommended by Miller's 7±2 heuristic and doesn't align with how users actually think. Persona research identified three primary intents: understanding the program (Persona B), exploring scientific content (Persona A), and finding tools and code (Persona C). We need an information architecture that maps the existing folders to user tasks, not file system structure.

## Decision

We adopt a 3-cluster information architecture mapping all 9 folders into three task-oriented groups:

- **Discover the Program** (01\_program\_overview, 08\_references) — "What is this?"
- **Explore the Science** (02\_xray\_modalities, 03\_ai\_ml\_methods, 04\_publications, 09\_noise\_catalog) — "What can I learn?"
- **Build and Compute** (05\_tools\_and\_code, 06\_data\_structures, 07\_data\_pipeline) — "How do I do it?"

The mapping is exhaustive (every folder assigned) and disjoint (no folder in two clusters). It is defined as a single constant in `explorer/lib/ia.py`.

## Consequences

### Positive
- Reduces top-level choices from 9 to 3, well within Miller's 7±2 range.
- Aligns navigation with user intent rather than file structure.
- Each cluster maps cleanly to a primary persona.
- The mapping is a single constant — easy to change if IA needs evolve.

### Negative / Trade-offs
- Some folders (e.g., 04\_publications) could arguably fit in multiple clusters. We chose based on primary use case.
- Users familiar with the file structure may need to learn the new navigation metaphor.
- The "Explore the Science" cluster is the largest (4 folders, ~145 notes), which may need sub-navigation.

### Follow-ups
- Implement sub-navigation within "Explore the Science" using folder-based categories (see sticky TOC in wireframe WF-002).
- Monitor analytics (if added) to see if users struggle to find content in the "wrong" cluster.
- Consider adding cross-cluster links in metadata panels for borderline content.

## Alternatives Considered

- **Flat 9-folder navigation**: Mirror the file system. Rejected because it doesn't match user mental models and exceeds cognitive load.
- **5-cluster grouping**: Finer-grained (e.g., separating "Publications" and "Noise Catalog" into their own clusters). Rejected because 5 top-level choices still exceeds the 3-choice simplicity that aligned with our 3 personas, and small clusters feel empty.
- **Dynamic clustering by tag**: Let tags define clusters at runtime. Rejected as too unpredictable — consistent IA requires a stable mapping.
