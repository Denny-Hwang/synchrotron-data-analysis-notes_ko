# PtychoNet: CNN 기반 타이코그래피 위상 복원(Ptychographic Phase Retrieval)

**참고문헌**: Guan et al. (2019)

## 개요

PtychoNet은 반복적 타이코그래피 위상 복원 알고리즘(ePIE, DM)을 학습된 합성곱 신경망(CNN)으로 대체하여 단일 패스 추론을 수행합니다 — 비교 가능한 이미지 품질을 유지하면서 약 90%의 속도 향상을 달성합니다.

## 문제 정의

### 반복적 위상 복원(Iterative Phase Retrieval) (전통적)

```
주어진: 회절 패턴 I_j = |F{P(r-r_j) × O(r)}|²

구하고자: 객체 O(r)와 프로브 P(r)

알고리즘 (ePIE):
  각 반복에 대해 (100-1000회):
    각 스캔 위치 j에 대해:
      1. ψ_j = P(r-r_j) × O(r)          # 출사파(Exit wave)
      2. Ψ_j = F{ψ_j}                    # 검출기로 전파
      3. Ψ'_j = √I_j × Ψ_j/|Ψ_j|       # 진폭을 측정값으로 대체
      4. ψ'_j = F⁻¹{Ψ'_j}               # 역전파
      5. O(r)와 P(r) 업데이트             # 그래디언트 업데이트
```

시간: 재구성당 수분에서 수시간.

### PtychoNet (딥러닝)

```
주어진: 단일 회절 패턴 I_j (256×256)

예측: 위치 j에서의 국소 객체 패치 (진폭 + 위상)

시간: 패턴당 밀리초
```

## 아키텍처

### 인코더-디코더 네트워크(Encoder-Decoder Network)

```
입력: 회절 패턴 (1ch, 256×256)
    │
    ▼ [인코더]
    ResBlock (64) → MaxPool
    ResBlock (128) → MaxPool
    ResBlock (256) → MaxPool
    ResBlock (512) → MaxPool
    │
    ▼ [병목층(Bottleneck)]
    ResBlock (512)
    │
    ▼ [디코더 — 진폭 분기]               ▼ [디코더 — 위상 분기]
    UpConv + skip (256)                   UpConv + skip (256)
    UpConv + skip (128)                   UpConv + skip (128)
    UpConv + skip (64)                    UpConv + skip (64)
    UpConv (1)                            UpConv (1)
    │                                     │
    ▼                                     ▼
    진폭 이미지 (256×256)                 위상 이미지 (256×256)
```

### 주요 설계 선택
- **이중 출력 헤드**: 진폭과 위상에 대한 별도 분기
- **잔차 블록(Residual Blocks)**: 더 나은 그래디언트 흐름, 더 깊은 네트워크 가능
- **스킵 연결**: 인코더의 고주파 세부 정보 보존
- **입력**: 회절 강도의 제곱근 (동적 범위 처리를 위해)

## 학습

### 데이터 생성
```
1. 실제 타이코그래피 데이터셋을 시뮬레이션하거나 수집
2. 반복법으로 재구성 (500회 이상 반복의 ePIE) → 정답(Ground Truth)
3. 학습 쌍: (회절 패턴, 국소 재구성 패치)
4. 데이터 증강: 회전, 노이즈 추가, 프로브 변형
```

### 손실 함수(Loss Function)

```python
# 진폭과 위상에 대한 결합 손실
loss = lambda_amp * MSE(pred_amp, gt_amp) + lambda_phase * MSE(pred_phase, gt_phase)

# 위상 래핑(Phase Wrapping) 처리 포함:
phase_diff = pred_phase - gt_phase
phase_diff_wrapped = torch.atan2(torch.sin(phase_diff), torch.cos(phase_diff))
phase_loss = MSE(phase_diff_wrapped, 0)
```

### 학습 매개변수
- **옵티마이저**: Adam, lr=1e-4
- **배치 크기**: 16-32
- **에포크 수**: 50-100
- **GPU**: 단일 NVIDIA V100/A100
- **학습 시간**: 4-12시간
- **학습 세트**: 10,000-50,000개 회절 패턴

## 성능

### 속도 비교

| 방법 | 재구성당 시간 | 상대적 |
|--------|----------------------|----------|
| ePIE (1000 반복, GPU) | 60 s | 1× |
| ePIE (100 반복, GPU) | 6 s | 10× |
| DM (500 반복, GPU) | 30 s | 2× |
| **PtychoNet (추론)** | **0.5 s** | **120×** |

### 품질 비교

| 지표 | ePIE (1000 반복) | PtychoNet |
|--------|-----------------|-----------|
| **SSIM (진폭)** | 참조 | 0.92-0.97 |
| **SSIM (위상)** | 참조 | 0.90-0.95 |
| **위상 오차 (rad)** | 0 (참조) | 0.05-0.15 |

*PtychoNet은 학습된 시료 유형에 대해 반복법의 품질에 근접합니다.*

## 강점

1. **속도**: 전체 반복 재구성보다 90-120배 빠름
2. **단일 패스**: 반복적 수렴 불필요
3. **병렬화 가능**: 각 패턴이 독립적으로 재구성됨
4. **실시간 호환**: 스트리밍 분석에 충분히 빠름

## 한계

1. **일반화**: 특정 시료 유형에서 학습; 새로운 유형에는 재학습 필요
2. **프로브 재구성 없음**: 알려진 프로브를 가정 (또는 별도 추정 사용)
3. **스티칭 아티팩트**: 전체 이미지를 위해 국소 패치를 스티칭해야 함
4. **학습 비용**: 대규모 반복 재구성 학습 세트가 필요
5. **정량적 정확도**: 위상 값이 정량적으로 정확하지 않을 수 있음

## 하이브리드 접근법: PtychoNet + 반복적 정제(Iterative Refinement)

최고의 결과는 DL 속도와 반복법의 정확도를 결합합니다:

```
회절 패턴
    │
    ├─→ PtychoNet (단일 순방향 패스)
    │       출력: 초기 재구성 (~0.5 s)
    │
    └─→ 반복적 정제 (PtychoNet 출력에서 시작하여 10-50 반복)
            출력: 정제된 재구성 (~3 s)
            품질: 1000-반복 ePIE와 거의 동일
```

이 하이브리드 접근법은 다음을 달성합니다:
- 전체 반복 재구성보다 10-20배 속도 향상
- 전체 반복법과 동등한 품질
- 양쪽의 장점: DL 속도 + 물리 기반 정확도
