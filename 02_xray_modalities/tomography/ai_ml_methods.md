# 토모그래피를 위한 AI/ML 방법

## 개요

토모그래피는 싱크로트론 과학에서 AI/ML이 가장 활발한 분야 중 하나입니다. 대규모 데이터 볼륨,
잘 정의된 수학적 모델, 명확한 품질 지표의 조합은 딥러닝 접근법에 이상적입니다. 주요 응용
분야에는 노이즈 제거, 재구성 가속, 분할, 아티팩트 제거가 포함됩니다.

## ML 문제 분류

| 문제 | 유형 | 입력 | 출력 |
|------|------|------|------|
| 노이즈 제거 | 이미지-이미지 | 노이즈 재구성 | 깨끗한 재구성 |
| 초해상도 | 이미지-이미지 | 저해상도 볼륨 | 고해상도 볼륨 |
| 분할 | 픽셀 분류 | 그레이스케일 볼륨 | 레이블 볼륨 |
| 아티팩트 제거 | 이미지-이미지 | 아티팩트 포함 | 깨끗한 이미지 |
| 희소 재구성 | 역문제 | 적은 투영 | 전체 볼륨 |
| 이상 검출 | 분류 | 슬라이스/영역 | 정상/비정상 |

## 노이즈 제거 방법

### TomoGAN

**아키텍처**: GAN 기반 노이즈 제거 (생성기: U-Net, 판별기: PatchGAN)

**학습**:
- 입력: 저선량(노이즈) 재구성
- 대상: 전체 선량(고품질) 재구성
- 손실: L1 + 적대적 + 지각적 (VGG 특징 매칭)

**결과**:
- SSIM 개선: 0.85 → 0.95+ (저선량 → 노이즈 제거)
- PSNR: +5-10 dB 개선
- 가우시안/중앙값 필터링보다 미세 구조 디테일을 더 잘 보존

**강점**: 지각적 품질, 선명한 엣지, 미세 디테일 보존
**약점**: 환각(hallucination) 가능성, 학습 중 모드 붕괴, 쌍 데이터 필요

*상세 리뷰 참조: [03_ai_ml_methods/denoising/tomogan.md](../../03_ai_ml_methods/denoising/tomogan.md)*

### Noise2Noise

**원리**: 자기지도학습 — 동일한 시료의 두 노이즈 관측으로 학습
- 깨끗한 대상 불필요
- 동일 조건에서 동일 시료의 두 번 스캔 필요

**APS에서의 응용**: 두 번의 빠른 스캔 수집 → 학습 쌍으로 사용
**한계**: 스캔 사이에 정적 시료 가정

*참조: [03_ai_ml_methods/denoising/noise2noise.md](../../03_ai_ml_methods/denoising/noise2noise.md)*

## 재구성 가속

### TomocuPy (GPU 가속 FBP)

- 표준 재구성 알고리즘의 CuPy 기반 GPU 구현
- TomoPy (CPU) 대비 동등 품질에서 20-30배 속도 향상
- GPU에서 FBP, gridrec, 반복법 지원
- GPU 메모리를 초과하는 대규모 데이터셋을 위한 스트리밍 파이프라인

*상세 분석 참조: [05_tools_and_code/tomocupy/](../../05_tools_and_code/tomocupy/)*

### 딥러닝 재구성

**FBPConvNet** 접근법:
1. 희소 각도에서 빠른 FBP 수행 (아티팩트 포함)
2. 학습된 CNN을 적용하여 아티팩트 제거 및 노이즈 제거
3. 결과: 적은 투영에서 고품질 재구성 (4-10배 선량 감소)

**암시적 신경 표현(INR)**:
- 볼륨을 연속 함수 f_θ(x,y,z)로 표현
- 측정된 투영에 맞추어 네트워크 파라미터 최적화
- 희소하고 불규칙한 각도 샘플링의 자연스러운 처리
- 4D를 위한 시간 차원 통합 가능: f_θ(x,y,z,t)

*참조: [03_ai_ml_methods/reconstruction/inr_dynamic.md](../../03_ai_ml_methods/reconstruction/inr_dynamic.md)*

## 분할

### 전통적 방법
- **오쓰(Otsu) 임계값**: 단순, 빠름, 고대비 데이터에 적합
- **워터셰드**: 접촉/겹침 객체용
- **랜덤 워커**: 그래프 기반, 저대비 처리
- **영역 성장**: 연결된 구조에 적합

### 딥러닝 분할

**3D U-Net**:
- 체적 의료/과학 이미지 분할을 위한 표준 아키텍처
- 스킵 연결이 있는 인코더-디코더
- 입력: 3D 패치 (64³~128³ 복셀)
- 출력: 복셀별 클래스 확률

**nnU-Net**:
- 자동 구성 프레임워크 — 최적 U-Net 구성을 자동으로 결정
- 2D, 3D, 캐스케이드 접근법 처리
- 많은 생의학 벤치마크에서 최첨단

**싱크로트론 데이터의 과제**:
- 극도로 큰 볼륨 (2048³ = 80억 복셀) — 패치 단위로 처리해야 함
- 제한된 레이블 학습 데이터 (수동 주석이 비쌈)
- 다른 시료/빔라인 간 도메인 시프트
- 클래스 불균형 (예: 큰 볼륨 내 작은 공극)

### DLSIA (과학적 이미지 분석을 위한 딥러닝)

ALS/LBNL에서 개발한 싱크로트론 데이터 전용 프레임워크:
- U-Net, 혼합 스케일 밀집(MSD) 네트워크 지원
- 의료가 아닌 과학적 이미지 분석용으로 설계
- MLExchange 플랫폼과 통합

## 아티팩트 제거

### 링 아티팩트 보정
- **원인**: 결함 검출기 픽셀이 재구성 이미지에 링을 생성
- **전통적**: 푸리에 필터링, 사이노그램 도메인 보간
- **DL 접근법**: CNN을 학습시켜 링 패턴을 식별하고 제거
- **장점**: DL은 아티팩트를 제거하면서 실제 원형 특징을 보존

### 금속 아티팩트 감소
- **원인**: 고원자번호 함유물이 줄무늬 아티팩트를 유발
- **DL 접근법**: 사이노그램 도메인에서의 인페인팅 + CNN 후처리
- **응용**: 금속 입자가 있는 환경 시료

## 자율 실험 조향

### 실시간 품질 평가
- CNN이 수집 중 재구성 품질을 평가
- 문제가 있는 스캔(모션 블러, 정렬 불량)을 즉시 표시
- 적응형 스캔 프로토콜 가능 (품질이 낮으면 더 많은 투영)

### 스마트 스캐닝
- ML이 최적 스캔 파라미터(에너지, 노출, 투영 수) 예측
- 선량 효율적 스캐닝 프로토콜을 위한 베이지안 최적화
- 고해상도 스캔을 위한 관심 영역 식별

## 현재 한계

1. **학습 데이터 부족**: 레이블된 싱크로트론 토모그래피 데이터셋이 거의 없음
2. **일반화**: 하나의 빔라인/시료 유형에서 학습된 모델이 전이되지 않을 수 있음
3. **3D 메모리 제약**: 전체 4096³ 볼륨이 단일 GPU에 맞지 않음
4. **환각 위험**: DL 노이즈 제거가 인위적 특징을 도입할 수 있음
5. **검증**: DL로 향상된 결과가 물리적으로 정확한지 어떻게 확인할 것인가?

## 개선 기회

1. **자기지도 방법**: 쌍 학습 데이터에 대한 의존도 감소
2. **파운데이션 모델**: 다양한 싱크로트론 데이터로 사전 학습, 작업별 미세 조정
3. **물리 정보 네트워크**: CT 순방향 모델을 네트워크 아키텍처에 통합
4. **연합 학습**: 원시 데이터 공유 없이 빔라인 간 학습
5. **불확실성 정량화**: 재구성과 함께 신뢰도 맵 제공
