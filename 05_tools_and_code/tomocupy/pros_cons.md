# TomocuPy 대 TomoPy -- 비교

## 성능 벤치마크

2048 x 2048 x 1500 사이노그램 데이터셋의 재구성:

| 메트릭 | TomocuPy (A100) | TomoPy (64코어 CPU) |
|--------|-----------------|----------------------|
| FBP 처리 시간 | ~25 s | ~180 s |
| 위상 복원 | ~3 s | ~25 s |
| 링 제거 | ~2 s | ~15 s |
| 최대 메모리 (장치) | ~18 GB VRAM | ~32 GB RAM |
| I/O 중첩 | 예 (비동기 DMA) | 제한적 (스레딩) |

## TomocuPy의 장점

- **속도** -- 해석적 재구성 방법에서 TomoPy보다 5-10배 빠르며, 빔라인에서 거의 실시간 피드백이 가능합니다.
- **스트리밍 I/O** -- 연산과 데이터 전송의 중첩으로 GPU를 바쁘게 유지하고 I/O 지연을 숨깁니다.
- **작은 코드베이스** -- TomoPy의 더 큰 C 확장 모듈보다 감사 및 수정이 더 쉽습니다.
- **CLI 호환성** -- APS 빔라인 데이터 관리 워크플로우에 직접 통합됩니다.
- **재현 가능한 벤치마크** -- 표준화된 시간 측정을 위한 `bench` 하위 명령을 포함합니다.

## TomocuPy의 단점

- **GPU 필수** -- CPU로 폴백할 수 없으며, NVIDIA 하드웨어와 작동하는 CUDA 툴킷이 필요합니다.
- **단일 GPU 전용** -- 매우 큰 데이터셋을 위해 여러 GPU에 분산할 수 없습니다.
- **제한된 알고리즘 세트** -- 해석적 (FBP) 방법만 지원; 반복적 솔버 (SIRT, CGLS, MLEM) 없음.
- **작은 커뮤니티** -- TomoPy보다 기여자 수가 적고 서드파티 문서가 적습니다.
- **CuPy 의존성** -- 잘 구성된 CUDA 환경이 없는 시스템에서는 CuPy 설치가 쉽지 않을 수 있습니다.

## TomoPy의 장점 (비교용)

- 성숙하고 문서화가 잘 되어 있으며 큰 사용자 커뮤니티를 보유.
- 해석적 및 반복적 알고리즘 모두 지원.
- CPU 전용 -- GPU 없이 모든 머신에서 실행.
- 플러그인 아키텍처로 ASTRA 및 기타 백엔드 허용.

## TomoPy의 단점 (비교용)

- 대규모 데이터셋에서 해석적 방법이 더 느림.
- C 확장을 소스에서 빌드하기 어려울 수 있음.
- 네이티브 GPU 가속 없음 (외부 플러그인 필요).

## 권장 사항

빠른 해석적 재구성이 우선이고 NVIDIA GPU가 사용 가능한 경우 TomocuPy를 사용하세요. 반복적 재구성, CPU 전용 환경 또는 더 넓은 알고리즘 선택이 필요한 경우 TomoPy로 전환하세요.
